# Rules for Writing the Results (Apresentação e Análise dos Dados)

## Purpose
This chapter presents and analyzes the data collected during the research. Each section must directly address one specific objective, using the theoretical framework from Chapter 2 to interpret findings.

## Core Principle: Mirror the Specific Objectives
The structure of this chapter is **directly derived from the specific objectives** listed in Chapter 1.

**Mapping rule:**
- Each specific objective → one main section in this chapter
- The section title = the specific objective, converted from verb-infinitive form to noun form (or by simply removing the initial verb)

**Example:**
| Specific Objective | Chapter 4 Section Title |
|---|---|
| a) Identify security systems available to the intranet | 4.1 Identification of security systems available to the intranet |
| b) Investigate security policies adopted by Organization X | 4.2 Investigation of security policies adopted by Organization X |
| c) Determine mechanisms used to reduce resistance | 4.3 Determination of mechanisms used to reduce user resistance |

## How to "Fill" Each Section

### Step 1: Present the Data
- Bring the treated/analyzed data collected in the field via the instrument described in Chapter 3
- Can include tables, charts, graphs, or direct quotations from interviewees
- Do NOT use two different visual formats (e.g., a table AND a chart) to show the same data — choose one

### Step 2: Analyze the Data Using Theory
- Connect the findings to the theoretical framework from Chapter 2
- The analysis means evaluating the data in light of what the authors say
- Cite the theory to validate, contrast, or contextualize the findings

### Analysis Pattern (Quantitative Example)
```
[Present the data/finding — e.g., "70% of respondents resist the periodic 
renewal of passwords."]

[Comment on significance — e.g., "This value is significant in terms of 
security risk, since..."]

[Bring in theoretical support — e.g., "According to Abc (2000), 'one of the 
greatest threats to Intranet security is the habit of not changing passwords regularly.'"]

[Conclude the analysis — e.g., "Therefore, security at Organization X faces 
serious risks in this aspect."]
```

### Analysis Pattern (Qualitative Example)
```
[Present an interviewee quote — e.g., "As stated by respondent E1, 'it is 
common for employees to resist the password renewal policy.'"]

[Interpret the data — e.g., "This suggests that the unit faces serious Intranet 
security risks."]

[Bring in theoretical support — e.g., "As Abc (2000) reminds us, one of the 
main threats to Intranet security relates to users not regularly changing passwords."]

[Draw a conclusion based on theory + data — e.g., "Thus, it is possible to 
affirm that security at Organization X faces significant risks in this dimension."]
```

## Rules for Tables, Charts, and Figures
- All visual elements must be accompanied by explanatory paragraphs
- Label all tables and figures with title and source
- Do NOT present a table and then a chart showing the same information
- Visual elements should be intercalated within the text, not grouped at the end

## Language and Style
- Maintain **impersonal academic tone**
- Always connect data to theory — never present "naked" data without interpretation
- Avoid opinions not grounded in data or theory
- Can innovate in format, but always contrast research results with what authors say

## Common Mistakes to Avoid
- Do NOT create sections that don't correspond to a specific objective
- Do NOT present data without analyzing it
- Do NOT analyze without referencing the theoretical framework from Chapter 2
- Do NOT use both a table and a graph for the same dataset
- Do NOT skip any specific objective — every objective must have a corresponding section
